Please note

Text typed for version 2.0 of the fonts will display legibly using version 2.5 (or 2.51), with only minor errors, but text typed for version 2.5 will not display acceptably using version 2.0 fonts.

There is an updated mapping and instructions in the package.

Changes from Ezra SIL 2.5

Ezra SIL 2.51

  • Word-final Zinor (U+05AE HEBREW ACCENT ZINOR) is always to the left. When it occurs word-medial, it is also to the left, unless that position is obstructed by a sin dot, holam or lamed, in which case it is centred. This is the same behaviour as for U+0599 HEBREW ACCENT PASHTA.
  • When U+0592 HEBREW ACCENT SEGOL and U+05A9 HEBREW ACCENT TELISHA QETANA occur word-medial, they are centred unless followed by U+200C ZERO WIDTH NON-JOINER, in which case they move to the left.
  • When U+05A0 HEBREW ACCENT TELISHA GEDOLA is word-medial, it is centred unless the consonant is preceded by U+200C ZERO WIDTH NON-JOINER, in which case it moves to the right.
  • There have been a few minor positional adjustments.

Ezra SIL SR 2.51

  • U+05AE HEBREW ACCENT ZINOR is always to the left, whether it is word-final or word-medial.
  • When U+0592 HEBREW ACCENT SEGOL, U+05A9 HEBREW ACCENT TELISHA QETANA and U+05A0 HEBREW ACCENT TELISHA GEDOLA occur word-medial, they are always centred.
  • There have been a few minor positional adjustments.

The Ezra SIL Hebrew Unicode Font package is available as a self-extracting archive, zip, or kmp (Keyman package) file. To download:

Ezra SIL 2.51 fonts release (Zip file)
NRSI staff, 2007-10-04
Download "EzraSIL251.zip", ZIP archive, 619KB [51596 downloads]

This download contains the fonts, documentation for installing the fonts, documentation for mapping files and mapping files. It does not contain any keyboards. In addition to downloading this font package, if you need a keyboard you can download the Keyman package below or another keyboard such as an Israeli keyboard layout (using Microsoft's MSKLC) which is available from here:  http://www.sbl-site.org/educational/biblicalfonts.aspx.

Ezra SIL 2.51 fonts release - Windows installer
NRSI staff, 2008-05-08
Download "EzraSIL251.exe", Windows application, 670KB [20110 downloads]

This package contains the same files as EzraSIL251.zip. This installer is only for Windows users. Once the package has been downloaded, you can double-click on it to install the font file and documentation. After installing, you can read the documentation files at Start / All programs / SIL / Fonts / EzraSIL. You will find the mapping files at C:\Program Files\SIL\Fonts\EzraSIL\mappings (unless you chose a different location).

Note that certain applications may not see the new font immediately. You may have to quit and restart the application for the font to become available.
Ezra SIL Unicode 2.4 keyboard and Ezra SIL 2.51 fonts (Keyman package)
NRSI staff, 2007-10-04
Download "EzraSIL251.kmp", Keyman keyboard package, 751KB [9559 downloads]

This package contains the same files as EzraSIL251.zip plus the Keyman keyboard and documentation files for the keyboard layout. The Keyman keyboard is version 2.4. It will work with Keyman 6 or 7.

This keyboard was developed for keyboarding biblical Hebrew. For guidelines on who can use Keyman and for downloading, go to:  http://www.tavultesoft.com. The “Ezra SIL Unicode 2.4” keyboard is designed to work with the Ezra SIL font which is a part of this package. However, you can use this keyboard with any Hebrew Unicode font.

If you download this package, you must have Keyman already installed. You can just double-click EzraSIL251.kmp to install the keyboard package and font. Once installed, documentation can be found at Start / All Programs / Ezra SIL Unicode 2.4. Other files (including documentation and mapping files) will be found here (or a similar path): C:\Documents and Settings\All Users\Application Data\Tavultesoft\Keyman\Keyboard\_Package\EzraSIL251.

This version of the keyboard contains a fix for the problem of fast typing reordering keystrokes when using Keyman with TSF. The basic effect of this is to make Keyman do all text input using TSF, rather than a mixture of TSF and ordinary keyboard events. If you use  FieldWorks you should upgrade to this version of the keyboard.

An Israeli keyboard layout (using Microsoft's MSKLC) is available from here:  http://www.sbl-site.org/educational/BiblicalFonts_SBLHebrew.aspx.

Hebrew and Greek Unicode Transliteration Package
Todd Hoatson and Christopher Samuel, 2007-06-27
Download "HebGrkUniTrans.zip", ZIP archive, 109KB [5683 downloads]

This package provides resources for Hebrew and Greek Unicode Transliteration. It comes with no claim for completeness and only limited documentation and support. We will try to act, however, on error reports.

The package contains a Keyman keyboard for Hebrew and Greek transliteration. Please read HebGrkTransUni.txt for an explanation of the scope and HebGrkTransUni.pdf for a list of keystrokes.

The package also contains a TECkit mapping which provides support for conversion between Hebrew Unicode & Transliterated Hebrew Unicode (Note that this conversion is not round-trip, i.e. converting from transliteration to Hebrew will not produce cantillation or punctuation, nor will it restore final-Shewa, the vowels in the divine name, or dagesh on initial consonants or after silent Shewa.) Disclaimer: This mapping is not perfect; no rule-based conversion could be. However, it achieves results at least as good as the change tables distributed with the SIL Ezra font.

If you are uncertain what to do with a .zip file, see Decompression Utilities, Downloading & Installation Tips. Within the archive is the keyboard, keyboard chart and mapping file.

Support

As these fonts and utilities are distributed at no cost, we are unable to provide a commercial level of personal technical support. This is the final release of these fonts.

Please note that these fonts are intended for use by experienced computer users. Installing and using these fonts is not a trivial matter. The most effective technical support is usually provided by an experienced computer user who can personally sit down with you at your computer to troubleshoot the problem.

Before requesting technical support, please:

  • Carefully read all the documentation provided with the fonts, including the documentation installed by the installation program.
  • Check out all the links on the web pages for these font packages, and read all the information and instructions they contain.
  • Review the list of Frequently Asked Questions to see if your question has already been answered. Additional information is also available on the general Font FAQ page.

If that fails to answer your question, or for more information, contact:
